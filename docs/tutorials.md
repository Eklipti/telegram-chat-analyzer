# Команды и утилиты

Справочник по всем командам CLI проекта.

Шаблон запуска:

```powershell
python main.py <команда> [аргументы]
```

По умолчанию:
- сырой экспорт ищется в `telegram/exports/raw_json/*.json` (берется самый новый файл);
- нормализованный файл ищется в `telegram/exports/processed_json/*.json` (берется самый новый; имена файлов — по хешу входного: `<hash>.json`);
- результаты пишутся в `output/YYYY_MM_DD_<hash>/`, где `<hash>` — первые 10 символов SHA256 входного JSON.

---

## Полный конвейер (all)

Запускает весь основной процесс: `normalize -> agg -> social -> html -> mobile -> excel`.
Что именно считается и какие метрики формируются — см. [`docs/features.md`](features.md).

### Команда запуска
```powershell
python main.py all
```

### Аргументы
* `--input` (опциональный): путь к сырому JSON-экспорту. Если не указан — берется самый новый файл из `telegram/exports/raw_json/`.
* `--force` (опциональный): перезаписать существующий нормализованный файл на шаге normalize.

### Краткий пример результата
После выполнения появятся (пути по умолчанию):
* `telegram/exports/processed_json/<hash>.json` (имя = первые 10 символов SHA256 входного файла)
* `output/YYYY_MM_DD_<hash>/agg/` (`all_aggregates.json` + `social_graph.json`)
* `output/YYYY_MM_DD_<hash>/report.html`
* `output/YYYY_MM_DD_<hash>/report.mobile.html`
* `output/YYYY_MM_DD_<hash>/report.xlsx`

## Нормализация (normalize)

Нормализует сырой JSON и сохраняет результат в `telegram/exports/processed_json/<hash>.json`, где `<hash>` — первые 10 символов SHA256 входного файла (тот же хеш используется для каталога вывода отчётов).

В корень нормализованного JSON добавляется поле `original_file_name` (например, `2026_01_week1[+3].json`).

### Команда запуска
```powershell
python main.py normalize
```

### Аргументы
* `--input` (опциональный): путь к сырому JSON. Если не указан — берется самый новый файл из `telegram/exports/raw_json/`.
* `--force` (опциональный): перезаписать существующий нормализованный файл.

### Результат
* `output/processed_json/<hash>.json`

## Контекстная история сообщений (context)

Утилита для создания текстовой истории сообщений за указанный период. Формирует хронологически упорядоченный список всех сообщений в простом текстовом формате. Поддерживает создание сжатой версии для оптимизации размера файла.

### Команда запуска
```powershell
python main.py context --date <дата_или_период>
```
*(Скрипт автоматически найдет нормализованный файл или создаст его из сырого JSON)*

### Аргументы
* `--date` (обязательный): дата или период для фильтрации сообщений:
    * `-1` — отчет за вчерашний день
    * `YYYY-MM-DD` — отчет за конкретный день (например, `2025-10-17`)
    * `YYYY-MM-DD_YYYY-MM-DD` — отчет за период (например, `2025-10-17_2025-10-20`)
* `--input` / `-i` (опциональный): путь к нормализованному JSON файлу.
* `--compress` (опциональный): создать дополнительно сжатую версию контекста.
* `--no-save` (опциональный): не сохранять **несжатую** версию (работает только с `--compress`).
* `--split` / `-s` (опциональный): разбить период на отдельные файлы для каждого дня (работает только с периодом).
* `--threads` / `-t` (опциональный): количество потоков для режима `--split` (по умолчанию 2, максимум 100).
* `--batch-size` (опциональный): размер батча сообщений для режима `--split` (по умолчанию 10000).
* `--min` (опциональный): минимальная длина сообщения для сжатой версии (по умолчанию 5).
* `--max` (опциональный): максимальная длина сообщения для сжатой версии (по умолчанию 250).

### Что делает `--compress`
Сжатие применяется к уже созданному файлу `context_<...>.txt` и формирует `context_<...>_compressed.txt`.

Это неархивирование, а очистка текста: сжатый файл меняет формат и содержимое, чтобы уменьшить шум и размер.

**Правила**:
- удаляются: 
  - пустые строки;
  - короткие сообщения (короче `--min` символов);
  - сообщения, состоящие только из упоминаний (`@username @user2 ...`);
  - сообщения, состоящие только из ссылок (`http://*`);
    - ссылки вида `http(s)://...` заменяются на `http://*`;
  - все эмодзи;
- из текста удаляются все спецсимволы, кроме букв/цифр/пробелов и `@ / * . , ! ? : ; -`;
- ограничиваются повторяющиеся символы (например, `??????` -> `??`);
- ограничиваются повторы одного слова подряд (например, `ха ха ха ха ха ха ха ха ха` -> `ха ха ха ха ха` (до 5 раз);
- последовательные дубликаты сообщений у одного автора не сохраняются;
- длинные сообщения обрезаются до `--max` символов;
- меняется формат вывода: временные метки убираются, а ник упрощается:
  - берется только первое слово ника;
  - из ника убираются круглые скобки и эмодзи;
  - из ника убираются `.,!?:;-` (если после очистки ник не становится пустым).

Если указан `--no-save`, то после создания сжатой версии **несжатый** `context_<...>.txt` удаляется.

### Результат
В папке `output/YYYY_MM_DD_<hash>/context/` создается:
* `context_<дата>.txt` — полная текстовая история.
* `context_<дата>_compressed.txt` — оптимизированная версия.
