# 📦 Telegram Chat Analyzer

Этот проект представляет интрумент для анализа чата по экспорту Telegram из Desktop приложения. 

## 🚀 Возможности

* Весь процесс разделен на шаги:
    1.  **Нормализация** (`step1`): Очистка данных, нормализация времени (опционально), обогащение (создание `media_cat`).
    2.  **Агрегация** (`step2`): Расчет всей статистики (топы, динамика) с использованием чистых словарей Python.
    3.  **Социальный граф** (`step3.5`): Углубленная статистика взаимодействий (упоминания, ответы, цитируемость) и контентный анализ (голосовые, ссылки, CAPS, форматирование, словарный запас).
    4.  **Отчетность** (`step3-4`): Генерация конечных отчетов для пользователя в html и Exel форматах.
* Запуск в CLI режими. *В будущем будет простой интерфейс на Tkinter.*
* **Три формата отчетов:**
    1.  **Desktop HTML:** Комплексный однофайловый отчет с кастомными интерактивными графиками.
    2.  **Mobile HTML:** Отдельная адаптивная версия с навигацией, оптимизированной для мобильных устройств *(в будущем будет улучшен)*.
    3.  **Excel (.xlsx):** Многостраничный отчёт с данными, готовыми для сводных таблиц (pivot-ready), микрографиками (sparklines) и условным форматированием (data bars).
* **Визуализация:** Графики созданы на чистом JavaScript и HTML Canvas API.

## 🛠️ Установка

0. Установите [Python](https://www.python.org/downloads) от 3.8+

1.  Клонируйте репозиторий:
    ```sh
    git clone https://github.com/Eklipti/telegram-chat-analyzer
    cd created
    ```

2.  Создайте и активируйте виртуальное окружение:

    ```sh
    python -m venv venv
    ```

    2.1 Если делаете в cmd:

        .venv\Scripts\activate.bat
        

    2.2 Если делаете в PowerShell:

        .\.venv\Scripts\Activate.ps1

3.  Установите зависимости:
    ```sh
    pip install -r requirements.txt
    ```

**Примечание:** Для анализа словарного запаса используется библиотека `pymorphy3` для лемматизации (совместима с Python 3.11+). При первом запуске может потребоваться несколько секунд для инициализации. Если библиотека недоступна, анализ продолжится без лемматизации с предупреждением.

## ⚙️ Использование

### 1. Подготовка данных

1.  Получите полный экспорт истории вашего Telegram-чата в формате **JSON**. Это можно сделать в Desktop приложении Telegram.
2.  Внутри папки проекта, создайте следующую структуру, если она не создана:
    ```
    Telegram Chat Analyzer/
    ├── telegram/
    │   └── exports/
    │       ├── raw_json/
    │       └── processed_json/
    ├── scripts/
    └── ...
    ```
3.  Поместите ваш JSON-файл (например, `result[+3].json`) в папку `telegram/exports/raw_json/`.

### 2. Запуск

Рекомендуемый способ — запустить весь конвейер одной командой. Скрипт автоматически найдет самый новый JSON-файл в папке `raw_json/`.

```sh
python main.py all
```

**Взаимодействия:**
* **Индекс цитируемости:** На чьи сообщения чаще всего отвечают (топ-10)
* **Матрица упоминаний:** Кого чаще всего упоминают через @username (топ-15)
* **Карта симпатий:** Кто кому чаще отвечает, исключая ответы самому себе (топ-20 пар)

**Контентный анализ:**
* **Любители голосовых:** Топ по суммарной длительности голосовых сообщений
* **Внешние ссылки:** Какие домены чаще всего упоминаются (YouTube, GitHub и др.)
* **Крикуны:** Кто злоупотребляет CAPS LOCK (процент сообщений заглавными буквами)
* **Оформители:** Топ по использованию форматирования текста (жирный, курсив, спойлеры)
* **Словарный запас (MATTR):** Лексическое разнообразие по методу Moving-Average Type-Token Ratio — среднее разнообразие в скользящем окне 1000 слов с лемматизацией и фильтрацией стоп-слов (минимум 1000 сообщений)

**Поведенческие паттерны:**
* **Скорость реакции:** Топ-10 самых быстрых пользователей по медианному времени ответа через reply (минимум 1000 сообщений)
* **"Совы" против "Жаворонков":** Категоризация пользователей по времени активности (кто пишет ночью 01:00-06:00, а кто днём 09:00-17:00, минимум 1000 сообщений)
* **Индекс "Самоцензуры":** Кто чаще редактирует свои сообщения после отправки (топ-10, минимум 1000 сообщений)

### 4. Дополнительные утилиты

**Утилита: Стенограмма по авторам (Author & Text)**

Специальная утилита для детального анализа контента. Она группирует все сообщения чата по авторам и сохраняет их в хронологическом порядке.

* **Команда запуска:**
    ```sh
    python main.py author_and_text
    ```
    *(Скрипт автоматически найдет нормализованный файл или создаст его из сырого JSON)*

* **Результат:** В папке `output/author_text/` создаются два файла:
    1.  `author_text_report.json` — структурированные данные для машинной обработки.
    2.  `author_text_report.txt` — читаемый текстовый отчет, который содержит:
        * **Топ пользователей** (рейтинг по количеству сообщений).
        * **Стенограмму:** полный список текстов сообщений для каждого пользователя, отсортированный от первого к последнему.

---

**Утилита: Профилирование JSON (Params)**

Опциональная утилита для инвентаризации исходного JSON-файла. Анализирует структуру данных, подсчитывает ключи, пути и типы данных, создает подробный Markdown-отчет.

* **Команда запуска:**
    ```sh
    python main.py params
    ```
    *(Скрипт автоматически найдет самый новый JSON-файл в папке `raw_json/`)*

* **Результат:** В папке `output/params/` создается файл `json_params.md` с детальной информацией о структуре JSON-экспорта.

---

**Утилита: Контекстная история сообщений (Context)**

Утилита для создания текстовой истории сообщений за указанный период. Формирует хронологически упорядоченный список всех сообщений в простом текстовом формате. Поддерживает создание сжатой версии для оптимизации размера файла.

* **Команда запуска:**
    ```sh
    python main.py context --date <дата_или_период>
    ```
    *(Скрипт автоматически найдет нормализованный файл или создаст его из сырого JSON)*

* **Аргументы:**
    * `--date` (обязательный): Дата или период для фильтрации сообщений:
        * `-1` — отчет за вчерашний день
        * `YYYY-MM-DD` — отчет за конкретный день (например, `2025-10-17`)
        * `YYYY-MM-DD_YYYY-MM-DD` — отчет за период (например, `2025-10-17_2025-10-20`)
    * `--input` / `-i` (опциональный): Путь к нормализованному JSON файлу. Если не указан, используется автоматический поиск.
    * `--compress` (опциональный): Создать дополнительно сжатую версию контекста
    * `--no-save` (опциональный): Не сохранять несжатую версию (работает только с `--compress`)
    * `--split` / `-s` (опциональный): Разбить период на отдельные файлы для каждого дня (работает только с периодом)
    * `--threads` / `-t` (опциональный): Количество потоков для параллельной обработки в режиме `--split` (по умолчанию 2, максимум 100)
    * `--batch-size` (опциональный): Размер батча сообщений для обработки в режиме `--split` (по умолчанию 10000 строк)
    * `--min` (опциональный): Минимальная длина сообщения для сжатой версии (по умолчанию 5)
    * `--max` (опциональный): Максимальная длина сообщения для сжатой версии (по умолчанию 250)

* **Примеры использования:**
    ```sh
    # Обычная версия
    python main.py context --date 2025-12-26

    # Обычная + сжатая версия (параметры по умолчанию)
    python main.py context --date 2025-12-26 --compress

    # Период с разбивкой на отдельные файлы для каждого дня (2 потока по умолчанию)
    python main.py context --date 2025-12-26_2025-12-29 -s

    # Период с разбивкой + 10 потоков (короткие алиасы)
    python main.py context --date 2025-12-01_2025-12-30 -s -t 10

    # Период с разбивкой + сжатая версия для каждого дня + 10 потоков
    python main.py context --date 2025-12-26_2025-12-29 -s --compress -t 10

    # Период с разбивкой + кастомный размер батча для экономии памяти
    python main.py context --date 2025-12-01_2025-12-30 --split --threads 5 --batch-size 5000

    # Только сжатая версия (без сохранения несжатой)
    python main.py context --date 2025-12-26 --compress --no-save

    # Период с разбивкой: только сжатые версии для каждого дня
    python main.py context --date 2025-12-01_2025-12-30 -s --compress --no-save -t 10

    # Сжатая версия с кастомными параметрами
    python main.py context --date 2025-12-26_2025-12-29 --compress --min 10 --max 500
    ```

* **Результат:** В папке `output/context/` создается:
    * `context_<дата>.txt` — полная текстовая история сообщений в формате:
        ```
        {отправитель} [{дата и время}]
        "{текст сообщения}"
        
        {отправитель} [{дата и время}]
        "{текст сообщения}"
        ```
    * `context_<дата>_compressed.txt` (при использовании `--compress`) — оптимизированная версия с:
        * Удалением коротких сообщений
        * Обрезкой длинных сообщений
        * Удалением эмодзи
        * Ограничением повторений символов и слов
        * Заменой ссылок на `http://*`
        * Статистикой сжатия в логах
    
    При использовании флага `--no-save` вместе с `--compress` несжатая версия не сохраняется — остается только сжатый файл `_compressed.txt`.
    
    При использовании `--split` с периодом создается отдельный файл для каждого дня: `context_2025-12-26.txt`, `context_2025-12-27.txt` и т.д. Обработка выполняется параллельно в нескольких потоках для значительного ускорения. По умолчанию используется 2 потока, но вы можете увеличить их количество флагом `--threads` (например, `--threads 10` для обработки 10 дней одновременно). Максимальное количество потоков ограничено 100.
    
    Для оптимизации использования памяти при обработке больших файлов сообщения обрабатываются батчами (по умолчанию 10000 сообщений на батч). Размер батча можно настроить флагом `--batch-size`.

    Сообщения отсортированы в хронологическом порядке.
